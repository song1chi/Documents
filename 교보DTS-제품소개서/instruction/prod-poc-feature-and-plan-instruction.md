### 문서 제목

자연어 DB 질의 PoC 검토를 위한 DeepCube.AI 핵심기능 소개 및 준비사항

### 1. 목적

고객에게 PoC 항목 제안을 위한 DeepCube.AI 제품의 핵심 기능 소개 및 준비사항 정리.
요청자 주문 내용: "작성한 문서를 기반으로 교보문고 측에서 해당 내용을 분석해 PoC 가능 여부를 확인할 수 있도록 하기 위함."

### 2. 목차

실제 시스템에서 정확히 어떤 세부기능을 제공하는지 기술된 세부 자료(상세 기능설명서)를 참조하여 작성한다.
산출물 목차는 다음과 같다.

1. 개요
2. PoC 핵심 기능
3. 고객 준비사항

본 PoC는 자연어 RDB 조회 기능 검증을 목적으로 한다.

### 3. 개요

PoC는 DeepCube.AI의 업무적합성을 검증하기 위한 것으로, 크게 3가지 기능을 검증한다.

- **시나리오 관리**: 시나리오 작성, 편집, 실행
- **자연어로 RDB 조회**: 자연어로 DB 조회하는 시나리오 작성, 편집, 실행 (검증해야 할 핵심 기능)

PoC 순서는 시나리오 관리 → 자연어 DB 조회 → API 연동이 적절하다.
자연어 DB 질의 시 DB 특성 및 고객 특성을 고려한 DB Agent (노드) 커스터마이징이 필요하다.

### 4. PoC 핵심 기능

#### 4.1 시나리오(워크플로우) 관리

시나리오 작성, 편집, 배포, 실행 기능

#### 4.2 자연어로 RDB 조회

- 아키텍처: DB Agent (노드) → LLM → MCP DB Server (연동 담당) → DB 서버
- DB Agent (노드) 커스터마이징: 품질 향상을 위해 고객 DB 특성에 맞추어 hint 등의 정보 제공 필요
- DB Agent (노드)가 포함된 시나리오 작성, 편집, 배포, 실행 기능 확인

### 5. 고객 준비사항

#### 5.1 LLM

ollama + local LLM (또는 public LLM 허용)
ollama는 대체 가능 (추가 연동 개발 필요)

#### 5.2 DeepCube.AI 실행 서버

1대: Ubuntu 24.04 LTS 또는 RHEL 10.x

#### 5.3 DB 서버

1안) PoC 전용 DB 서버 — 고객 실 DB의 복제
2안) PoC 전용 schema/view (기존 DB ß서버)

접속 계정은 각 schema/view에 대한 조회권한만 부여.

#### 5.4 자연어 질의 예문

고객 DB에 대한 자연어 질의 예문 (많을수록 좋음)

## 솔루션 개요

### 기업 AI, RAG(문서검색)로는 부족합니다

RAG 기반 문서 검색은 **정책·매뉴얼**은 답할 수 있어도, 기존 RDB에 들어있는 **실시간 정보**(매출·재고·고객 현황 등)은 답하기 어렵습니다.  

조직 및 기업에 필요한 것은  **문서와 RDB를 아우르는 AI**를 만들어, 기존 시스템(RDB 포함)과 연결되면서, 조직이 통제하며 운영하는 것입니다.

**DeepCube.AI**는  문서뿐 아니라 RDB를 포함한 정보 처리에 대하여 **설계 · 연결 · 운영**을 하나의 플랫폼에서 제공하는 엔터프라이즈 AI 솔루션입니다.

### DeepCube.AI 한눈에

| | |
|:---|:---|
| **무엇인가** | AI 업무 시나리오를 시각적으로 설계·실행·관리하는 **엔터프라이즈 AI 플랫폼** |
| **어떻게 다른가** | **사내 문서(RAG)** 와 **실시간 정보(업무용 RDB 연동 및 조회)** 를 모두 다루는 에이전트형 AI |
| **누가 쓰는가** | 현업·기획 담당자(설계), IT·운영 담당자(관리), 내외부 시스템(웹 포탈 및 그룹웨어 등과 연동) |
| **어디에 쓰는가** | 지식 검색, 업무 자동화, 데이터 기반 응답, 고객·직원 지원, 내외부 시스템에 AI 연동 |

### 설계 · 배포 · 실행 · 연동

```
    [  설계  ]    →    [  배포 ]    →    [  실행  ]    →    [  연동  ]
    시각 에디터    →     운영 콘솔     →      AI 엔진     →   내외부 시스템
       └──────────────────┴─────────────────┴──────────────────┘
                                   │
                       사내 문서 · 업무 DB · AI 모델
```

**설계** — 문서·DB·검색·LLM을 조합하고, 테스트 실행으로 노드별 진행·입출력을 실시간 검증      
**배포** - beta·prod 환경 분리, 검증·승인 후 운영 반영, 롤백·감사로 배포 통제  
**실행** — 운영 배포 후 end user의 대규모 요청을 동시 처리(비동기·병렬)하여 안정적인 실행  
**연동** — 내외부 앱·포털에 API로 AI 서비스 제공  

기업형 인프라에서 운영:
**폐쇄망** - 폐쇄망 내 구성된 기업 자체 LLM(로컬 LLM), RAG, 기존 RDB 등과 연동
**보안** - 사람·권한·시나리오·모델을 중앙에서 통제

### 핵심 역량

#### 시각적 AI 워크플로우 - 노코드 GUI
**복잡한 AI 업무를, 누구나 설계**

드래그 앤 드롭 에디터로 개별 기능을 가진 노드를 조합하여 업무 흐름을 구성합니다.  
설계 → 테스트 → 배포 → 운영까지 **개발팀 없이** 한 플랫폼에서 이어집니다.

#### 문서와 RDB, 한 플랫폼에서
**RAG와 실시간 조회를 함께**

| | 문서 | DB |
|:---|:---|:---|
| **다루는 것** | 정책, 매뉴얼, 보고서, 계약서 | 매출, 재고, 고객, 거래·업무 현황(기존 RDB 연동) |
| **방식** | 벡터 검색(RAG) | 실시간 조회 |
| **가치** | 사내 지식·규정 기반 Q&A | 질문하는 **그 순간**의 실제·최신 값 |

문서만으로 답할 수 없는 **현재 수치**와, DB만으로는 담기지 않는 **규정·가이드**를 **한 시나리오에서 조합**합니다.

지원 RDBMS: Oracle, MySQL, PostgreSQL 등 주요 RDBMS   

#### Human-in-the-loop — RDB 조회, 실행 전·실행 중 사람이 검증 가능

Text-to-SQL로 생성된 쿼리를 **자동 실행하지 않고**, 담당자가 검토·승인한 뒤 실행합니다.

1. **질의 적정성 검증** — 자연어 질의가 업무·권한 범위에 맞는지 확인
2. **SQL 미리보기·수정** — 생성된 SQL을 담당자가 검토하고 필요 시 수정
3. **승인 워크플로우** — 실행 중인 워크플로우에 대한 승인 후 다음 단계 진행
4. **실행** — 승인된 SQL만 RDB에 반영

> 규제 산업(금융·공공)이 요구하는 **감사 가능 구조** — 누가·언제·어떤 SQL을 승인·실행했는지 추적

#### 하나의 플랫폼, 조합 가능한 AI
**필요한 역량만 골라 쓰는 모듈형 구성**

| 역량 | 비즈니스 가치 |
|------|---------------|
| **문서 Q&A (RAG)** | 사내 문서·정책·매뉴얼 검색·답변 |
| **DB 실시간 조회** | 매출·고객·재고 등 **지금 이 순간**의 실제 값 |
| **다중 LLM** | 로컬 LLM 및 OpenAI, Anthropic, Upstage, Google, Ollama 등 지원 — 조직 정책에 맞는 모델 선택 |
| **웹 검색** | 뉴스·공시·시장 동향 등 **최신 외부 정보** 반영 |
| **복합 자동화** | 문서 확인 → DB 조회 → 초안 작성 등 **다단계 업무**를 한 시나리오로 |

#### 엔터프라이즈급 보안 및 운영
**도입 이후가 더 중요하기 때문에**

- **접근 권한 관리** — 부서·역할(RBAC)·시나리오 단위로 누가 무엇을 쓸 수 있는지 명확히 한다.
- **시나리오 운영** — 카테고리·공유·즐겨찾기·운영 배포로 AI 자산을 조직의 공용 자산으로 만든다.
- **감사·추적** — 사용 이력을 분리 저장·조회해 내부 통제·규제 대응
- **온프레미스** — 데이터가 사내 인프라에 머무는 배포

### 이런 업무가 바뀝니다

**지식 검색·Q&A**  
내부 정책·매뉴얼 질의에 즉시 응답 — **벡터 검색(RAG)** 으로 사내 문서와 연결

**DB·업무시스템 연계**  
“매출은?” “고객 현황은?” — DB **실시간 조회**로 **실제·최신** 데이터 즉시 응답

**문서 + DB 복합 응답**  
“이 규정에 따른 우리사 현황은?” — **문서에서 규정 확인 → DB에서 현황 조회**를 한 흐름으로

**업무 문서·보고서**  
입력·DB 데이터를 반영한 초안 생성 — 반복 작성 시간 절감

**실시간 정보 요약**  
뉴스·공시·동향 반영 — 웹 검색과 AI 요약을 한 흐름으로

**고객·직원 지원**  
포털·앱·콜센터에 AI 삽입 — 기존 채널을 그대로 확장

> DeepCube.AI는 **설계(시각 에디터) · 관리(운영 콘솔) · 실행(AI 엔진) · 연동(API)** 을 하나로 제공하는 엔터프라이즈 AI 플랫폼입니다.

## 솔루션 특장점

### DeepCube.AI가 다른 다섯 가지 이유

#### 01 · 현업이 직접 설계한다
개발 의존 없이 화면에서 AI 업무를 설계·테스트합니다. **만들고 쓰는 사람이 같은 조직** 안에 있을 수 있습니다.

#### 02 · 문서와 DB를 모두 다룬다
사내 문서(RAG)와 업무 DB(실시간 조회)를 **한 시나리오에서 조합**합니다. 챗봇 하나로는 닿기 어려운 **업무 전체 맥락**을 다룹니다.

#### 03 · 조직이 통제한다
권한·시나리오·감사 로그를 **운영 콘솔 한곳**에서 관리합니다. AI를 도입한 뒤에도 **누가 무엇을 쓰는지** 통제할 수 있습니다.

#### 04 · 기존 시스템에 녹아든다
별도 챗봇 UI 없이 **표준 API**로 포털·백오피스·앱에 AI를 삽입합니다. **이미 쓰는 화면**을 그대로 확장합니다.

#### 05 · 기업 환경에 맞게 배포한다
**온프레미스** 설치와 인증·감사 체계로 데이터·망 정책을 지켜야 하는 조직에 맞춥니다.

### Before → After

| 기존 | DeepCube.AI 도입 후 |
|------|---------------------|
| AI는 개발팀 전용 | **현업이 직접** 설계·운영 |
| 문서만, 또는 DB만 | **문서와 DB**를 한 플랫폼에서 |
| 챗봇 하나 띄우고 끝 | **업무 시스템과 연결**된 AI |
| 사용 이력·권한 불명확 | **권한 · 감사**로 통제 가능 |

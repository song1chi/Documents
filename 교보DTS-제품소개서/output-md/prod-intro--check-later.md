## 솔루션 개요

### 기업 AI, ‘챗봇 하나’로는 부족합니다

문서 검색 챗봇 하나로는 **정책·매뉴얼**은 답할 수 있어도, **지금 이 순간의 매출·재고·고객 현황**은 답하기 어렵습니다.  
반대로 DB만 연결하면 **규정·가이드** 등의 업무 문서는 다루기 어렵습니다.


진짜 과제는 **문서와 DB를 아우르는 AI**를 만들고, 기존 시스템과 연결하고, 조직이 통제하며 운영하는 것입니다.

**DeepCube.AI**는 **설계 · 연결 · 운영**을 하나의 플랫폼에서 제공하는 엔터프라이즈 AI 솔루션입니다.

### DeepCube.AI 한눈에

| | |
|:---|:---|
| **무엇인가** | AI 업무 시나리오를 시각적으로 설계·실행·관리하는 **엔터프라이즈 AI 플랫폼** |
| **어떻게 다른가** | **사내 문서(RAG)** 와 **업무 DB(실시간 조회)** 를 모두 다루는 에이전트형 AI |
| **누가 쓰는가** | 현업·기획 담당자(설계), IT·운영 담당자(관리), 내외부 시스템(웹 포탈 및 그룹웨어 등과 연동) |
| **어디에 쓰는가** | 지식 검색, 업무 자동화, 데이터 기반 응답, 고객·직원 지원, 내외부 시스템에 AI 연동 |


### 설계 · 관리 · 실행 · 연동

```
    [  설계  ]    →    [  관리  ]    →    [  실행  ]    →    [  연동  ]
    시각 에디터    →     운영 콘솔     →      AI 엔진     →   내외부 시스템
       └──────────────────┴─────────────────┴──────────────────┘
                                   │
                       사내 문서 · 업무 DB · AI 모델
```

**설계** — 문서·DB·검색·LLM을 화면에서 조합  
**관리** — 사람·권한·시나리오·모델을 중앙에서 통제  
**실행** — 진행 상황을 실시간으로 확인하며 안정적으로 처리  
**연동** — 내외부 앱·포털에 API로 AI 서비스 제공

### 핵심 역량

#### 시각적 AI 워크플로
**노코드 GUI로 복잡한 AI 업무를 누구나 설계**

드래그 앤 드롭 에디터로 업무 흐름을 구성합니다.  
설계 → 테스트 → 배포 → 운영까지 **개발팀 없이** 한 플랫폼에서 이어집니다.

**비즈니스 가치**

- **IT 개발 의존 제로** — 코딩 없이 현업 담당자가 직접 AI 시나리오 설계·수정
- **비용·시간 절감** — 개발 요청 → 대기 → 구현 사이클 제거, 즉시 설계·테스트
- **빠른 시행착오** — 현업이 직접 테스트하며 개선, 변화하는 업무에 신속 대응

**차별화된 GUI 기능**

- **플로우 설계** — 문서 검색 → LLM 판단 → DB 조회 → 응답 생성 등 복잡한 흐름을 블록으로 연결
- **실시간 테스트** — 설계 화면에서 즉시 실행·확인, 각 단계 입출력을 눈으로 확인

```
[ 사용 흐름 예시 ]

1. 현업이 에디터에서 시나리오 설계
   (문서 검색 노드 + DB 조회 노드 + LLM 응답 노드 연결)
      ↓
2. 테스트 버튼 클릭 → 실시간 결과 확인
   (각 노드별 입출력 데이터 즉시 표시)
      ↓
3. 수정 → 재테스트 (반복)
      ↓
4. 배포 진행 
   (검수/인수 테스트, 보안/권한 설정, 내부 승인, 운영 환경 적용)
      ↓
5. 사용자에게 API/UI로 서비스 제공
```

#### 문서와 DB, 한 플랫폼에서
**RAG와 실시간 조회를 함께**

| 항목 | RAG (문서) | RDB (DB 직결) |
|:---|:---|:---|
| **데이터 유형** | 문서, 정책, 매뉴얼, 보고서, 계약서 | 매출, 재고, 고객, 거래 등 구조화 데이터 |
| **접근 방식** | 벡터 검색 (유사도 기반) | SQL 직접 조회 (정확한 값) |
| **최신성** | 색인 갱신 주기 (분~시간 단위) | **실시간 (초 단위)** |
| **정확성** | 유사 문서 검색 | **정확한 수치** 제공 |
| **업무 적용** | 지식·규정 기반 Q&A | **의사결정용 실시간 수치** |
| **가치** | 사내 지식·규정 검색·답변 | 질문하는 **그 순간**의 실제·최신 값 |

문서만으로 답할 수 없는 **현재 수치**와, DB만으로는 담기지 않는 **규정·가이드**를 **한 시나리오에서 조합**합니다.

**RDB 직결형 아키텍처**

DeepCube.AI는 업무 DB를 RAG 벡터로 변환하지 않고 **직접 연결**합니다.

- **Text-to-SQL 자동 생성**  
  "이번 달 매출은?" 같은 자연어 질문을 SQL 쿼리로 자동 변환.  
  사용자는 SQL을 몰라도 데이터 조회 가능.  
  복잡한 조인·집계·필터 조건도 LLM이 생성.

- **다양한 RDB 지원**  
  Oracle, MySQL, PostgreSQL, MS SQL Server, MariaDB 등 주요 RDBMS 연동.  
  JDBC/ODBC 표준 프로토콜 지원으로 기존 DB 스키마 그대로 연결.

- **실시간 데이터 정합성**  
  **지금 이 순간**의 최신 값을 DB에서 직접 조회.  
  RAG는 벡터 색인 갱신 주기(분~시간 단위) 동안 시점 지연 발생.  
  트랜잭션 데이터·실시간 현황도 즉시 반영.

#### 하나의 플랫폼, 조합 가능한 AI
**필요한 역량만 골라 쓰는 모듈형 구성**

| 역량 | 비즈니스 가치 |
|------|---------------|
| **문서 Q&A (RAG)** | 사내 문서·정책·매뉴얼 검색·답변 |
| **DB 실시간 조회** | 매출·고객·재고 등 **지금 이 순간**의 실제 값 |
| **다중 LLM** | OpenAI, Anthropic, Upstage, Google, Ollama 등 — 조직 정책에 맞는 모델 선택 |
| **웹 검색** | 뉴스·공시·시장 동향 등 **최신 외부 정보** 반영 |
| **복합 자동화** | 문서 확인 → DB 조회 → 초안 작성 등 **다단계 업무**를 한 시나리오로 |

#### 기업형 인프라
**도입 이후가 더 중요하기 때문에**

**① AI 엔진 — 고성능 처리 아키텍처**

- **대규모 동시 요청 처리**  
  수십~수백 명의 동시 사용자 질의를 안정적으로 처리.  
  비동기·병렬 처리 엔진으로 응답 지연 최소화.

- **응답 시간 최적화**  
  LLM 추론·벡터 검색·DB 조회를 병렬 실행.  
  불필요한 대기 시간 제거, 사용자 경험 개선.

**② 보안 체계 — 데이터 유출 원천 차단**

- **로컬 LLM 지원**  
  OpenAI·Anthropic 등 외부 API 대신 **사내 LLM 서버** 연동 가능.  
  질문·문서·DB 데이터가 **외부로 나가지 않음** → 자료 유출 원천 차단.  
  온프레미스 배포로 데이터가 사내 인프라에 머물도록 설계.

- **접근 통제**  
  부서·역할·시나리오 단위로 세분화된 권한 관리.  
  시나리오별·데이터 소스별 접근 제어.  
  SSO/LDAP 연동으로 기존 인증 체계와 통합.

- **감사 및 추적**  
  누가·언제·어떤 시나리오를·어떤 데이터로 실행했는지 **전체 이력 기록**.  
  분리 저장·조회로 내부 통제·규제 대응.  
  민감 정보 접근 이력 추적 가능.

**③ 권한 관리 — 부서별 데이터 분리**

- **역할 기반 접근 제어 (RBAC)**  
  관리자·설계자·사용자 역할 분리.  
  시나리오·데이터·모델별 세분화된 권한 설정.

- **부서별 권한 분리**  
  인사 데이터는 인사팀만, 재무 데이터는 재무팀만 접근.  
  교차 접근 원천 차단으로 정보 유출 방지.

- **승인 절차**  
  민감 시나리오 배포 시 승인 프로세스 적용.  
  변경 이력 추적으로 언제·누가·무엇을 수정했는지 확인.

- **시나리오 운영**  
  카테고리·공유·즐겨찾기·환경별 배포로 AI 자산을 조직 자산으로 관리.

### 이런 업무가 바뀝니다

**지식 검색·Q&A**  
내부 정책·매뉴얼 질의에 즉시 응답 — **벡터 검색(RAG)** 으로 사내 문서와 연결

**DB·업무시스템 연계**  
“매출은?” “고객 현황은?” — DB **실시간 조회**로 **실제·최신** 데이터 즉시 응답

**문서 + DB 복합 응답**  
“이 규정에 따른 우리사 현황은?” — **문서에서 규정 확인 → DB에서 현황 조회**를 한 흐름으로

**업무 문서·보고서**  
입력·DB 데이터를 반영한 초안 생성 — 반복 작성 시간 절감

**실시간 정보 요약**  
뉴스·공시·동향 반영 — 웹 검색과 AI 요약을 한 흐름으로

**고객·직원 지원**  
포털·앱·콜센터에 AI 삽입 — 기존 채널을 그대로 확장

> DeepCube.AI는 **설계(시각 에디터) · 관리(운영 콘솔) · 실행(AI 엔진) · 연동(API)** 을 하나로 제공하는 엔터프라이즈 AI 플랫폼입니다.

## 솔루션 특장점

### DeepCube.AI가 다른 다섯 가지 이유

#### 01 · 현업이 직접 설계한다
**노코드 GUI**로 개발 의존 없이 화면에서 AI 업무를 설계·테스트합니다.  
드래그 앤 드롭 플로우 설계, 실시간 테스트, 버전 관리·롤백 기능 제공.  
**만들고 쓰는 사람이 같은 조직** 안에 있을 수 있습니다.

**TCO 절감** — 개발 요청 → 대기 → 구현 사이클 제거. IT 개발 비용·시간 대폭 감소.

#### 02 · 문서와 DB를 모두 다룬다
사내 문서(RAG)와 업무 DB(실시간 조회)를 **한 시나리오에서 조합**합니다.  
**Text-to-SQL** 기술로 자연어 질문을 SQL로 자동 변환 — 사용자는 SQL 몰라도 DB 조회 가능.  
**실시간 데이터 정합성** — 질문하는 **그 순간**의 최신 값으로 의사결정 지원.

챗봇 하나로는 닿기 어려운 **업무 전체 맥락**(규정+현황)을 다룹니다.

#### 03 · 조직이 통제한다
권한·시나리오·감사 로그를 **운영 콘솔 한곳**에서 관리합니다.

**로컬 LLM 지원** — 외부 API 의존 없이 사내 LLM 서버 연동. 질문·문서·DB 데이터가 **외부로 나가지 않음**.  
**부서별 권한 분리** — 인사 데이터는 인사팀만, 재무 데이터는 재무팀만 접근. 교차 접근 차단.  
**감사 로그** — 누가·언제·무엇을 실행했는지 전체 이력 기록. 내부 통제·규제 대응.

AI를 도입한 뒤에도 **누가 무엇을 쓰는지** 통제할 수 있습니다.

#### 04 · 기존 시스템에 녹아든다
별도 챗봇 UI 없이 **표준 API**로 포털·백오피스·앱에 AI를 삽입합니다. **이미 쓰는 화면**을 그대로 확장합니다.

#### 05 · 기업 환경에 맞게 배포한다
**온프레미스** 설치와 인증·감사 체계로 데이터·망 정책을 지켜야 하는 조직에 맞춥니다.

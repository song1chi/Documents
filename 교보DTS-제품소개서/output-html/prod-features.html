1. 문서 목적·범위

1.1 목적

본 문서는 교보디티에스(주) 및 교보그룹·관계사가 DeepCube.AI가 실제로 제공하는 기능을 기능 단위로 파악하고, PoC(개념검증) 범위·기술 적합성·선행 조건을 판단할 수 있도록 작성되었습니다.

1.2 범위

구분 포함 비고
표준 설치 모듈 dc-admin, dc-editor, dc-editor-api, dc-flow, dc-log PoC 기본 범위
선택 모듈 dc-rag-api, dc-rag-front (RAG Builder) 표준 매니페스트 보류 — 별도 협의
로드맵 항목 PostgreSQL 전환, 인증 고도화 현황·리스크로 명시

10. I. 외부 연동 API (B2B)

10.1 표준 연동 시퀀스

[외부 시스템]
    │
    ├─(1) POST /ui-api/auth/token  (x-navi-key)
    │       └─ JWT access token
    │
    ├─(2) GET /ui-api/scenario/my/list  (Bearer)
    │       └─ scenarioId 목록
    │
    ├─(3) POST /flow/query  (Bearer, scenario_id, question)
    │       └─ 202, query_task_id
    │
    └─(4) GET /flow/stream/{query_task_id}  (SSE)
            └─ 단계별 data + 최종 결과

10.2 주요 API 엔드포인트

구분 Method Path 인증 설명

인증 GET /ui-api/auth/verify Bearer (선택) 토큰 검증
시나리오 GET /ui-api/scenario/my/list Bearer 내 시나리오 목록
실행 POST /flow/query Bearer (nginx) 비동기 실행
실행 GET /flow/stream/{task_id} Bearer (nginx) SSE 스트리밍
